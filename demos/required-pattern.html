4. required / pattern / type 検証を突破

HTML5 のネイティブ検証（required、pattern、type="email"）は 便利ですが、フォームに novalidate を付けたり属性を消すだけで無効化できます。

突破の理屈: 最初の送信はネイティブ検証でブロックされます（＝UXとしては有効）。 しかし form.noValidate=true や required/pattern 属性の削除で 検証は消え、不正値でも送信されます。curl ならそもそも検証は存在しません。

正しい対策

メール形式・必須・書式はサーバーでも必ず再検証する。クライアント検証は入力補助と割り切る。
